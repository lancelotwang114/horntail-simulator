## 1. 專案是什麼

楓之谷「闇黑龍王（Horntail）」攻略走位**互動模擬器**。在地圖底圖上，用可切換的「階段」呈現各職站位移動、目標部位（〇）、已斷部位（✕）、危險／安全／下煙區域。最終目標：做成**多人即時房間**，host 開房、大家跟著看走位。

## 3. 攻略階段資料（目前 7 階段）

0. 開場・就定位
1. 第一階段・斷尾（27 分刀賊下煙 30 秒×2，除弩手槍神外集火尾；60% 未斷弩手留手；記錄斷尾血量%）
2. 第二階段・左手（英雄轉左）
3. 第三階段・左頭+中頭（中頭 50% 黑鎖，主 CALL 計次；黑鎖時標賊弩手靠右）
4. 第四階段・右手→右頭+中頭（右頭<50% 狂暴下煙）
5. 第五階段・狂暴下煙（集火右頭+中頭+右手；怕電退最右安全區）
6. 最後階段・收尾雙腿（英雄右下、拳霸左上、箭神左下打左腿）

**鐵則：尾巴一定要最先斷，其他部位先斷=挑戰失敗；肚子最後打。**
官方擊破順序：尾→左頭→中頭→右頭，肚子最後。

## 4. 資料模型（已成形）

每張板 = `board`，每階段含：
```
{ name, color, notes[[class,text]...],
  pos:{ roleKey:[x%,y%] },          // 職業站位
  marks:[{kind:'O'|'X', x,y, r}],   // 目標圈/已斷叉，可多個
  zones:[{kind:'smoke', x,y,w,h,t}] // 下煙區（每階段）
}
```
另有 **GLOBAL_ZONES**（4 紅危險區 + 1 綠安全區），固定每階段顯示、鎖定不可編輯。
座標皆為地圖百分比。編輯器可匯出此 JSON（之前已用此流程定版多次）。

職業 key 對照：hero=英雄, long=龍騎, dao=刀賊, biao=標賊, nu=弩手, qiang=槍神,
gong=箭神, fa=冰雷, quan=拳霸；分身 hero2/fa2/long2（英雄2/冰雷2/龍騎2）。

## 5. 多人版的決策（已敲定）

- **連線**：用 **Trystero**（免自架 signaling，GitHub Pages 可直接跑；不穩再換策略或退 PeerJS）
- **控制權**：預設 host 控制，**可交棒**給某位觀眾（authority token 機制）
- **觀眾視角**：**強制跟隨**控制者的當前畫面（不自由翻看）
- **host 斷線**：預設通知大家「房間結束」，先不做自動轉移
- **持久化**：純 P2P 不做釘板（關掉房間即消失）；要長期收藏另需儲存層（分享連結或 Supabase）
- **託管**：GitHub Pages（靜態），repo 名建議 `horntail-simulator`
